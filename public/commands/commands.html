Commands:

Classic terminal commnads

Being able to route in your terminal can be a very handy thing to do, which can save you alot of time and work

The most used commands are as follows

  • ls
  • cd
  • mkdir
  • rmdir

"ls" stands List Files. When this command is given in the terminal, the terminal will list all files in the directory you are currently in

"cd" stand for change directory. There are different ways to use cd. If you type cd and hit enter, your terminal will go to the root directory. If you type cd ../ your terminal will jump back one directory. If you type example cd movies you terminal will change to the movies directory, if there is such a directory avabile

"mkdir" stands for make directory. This is very self explanatory.

"rmdir" stands for remove directory. This is also very self explanatory

Node

To install and use node we use commands in our terminal. Examples of these commands are the npm (Node Package Manager)

The most common command to run is the npm install. When we run "npm install" we get all our dependencies in our package.json. These dependises are then added to our node_modules which is a folder with the same name i.e "node_modulers", which can we found at our library root

We use npm run to run scripts from our package.json file. The scripts can for example be which port to use when we run our application or if we want to run the application in a dev mode


nodemon is utility til will monitor for any changes in our source and restart the server when sensing a change to the code. We therefore use nodemon when we develop our apps

In order to use nodemon we use our Node Package Manger and in our terminal run "npm install -g nodemon" (the "-g" stand for gloabally)

We can then serve our app with nodemon by either type "nodemon app.js" to the file we want to run or make a script on our package.json like this: "startdev": "cross-env PORT=3000 nodemon app.js

So in other words, we use node to launch our applications and nodemon when we develop them

Last option is to also use budo, which we can use if we wish to launch html files and be able to see the changes we make efficiently, with a live-reload. To do so we use budo-commands. This is good because budo doesn't care about back-end changes and doesn't update those, but only focues on our html and css.

We run budo with pnx so it doesn't run globally like this:

npx budo app.js

If you for example wish live reload you add "--live" after your command like "npx budo app-js --live"


Express

We use Express to serve our webapplications.

Since Express is a web framework for Node.js we have to install first. To do so we run the following command in our terminal:

"$ npm install express --save"

We can now access Express to serve our application

In order to do so we have to import Express into our initial js file like this: const express = require("express")

We can then parse express with initialization express to a consant and run the follow code

const app = express();

app.use(express.json());

app.use(express.static('public'));

Also we need to make express listen a custom port, in our case port 3000

const port = process.env.PORT ? process.env.PORT : 3000;

const server = app.listen(port, (error) => {
if (error) {
console.log('something went wrong with starting the server', error); }

console.log('Server is running on port', server.address().port);
});

Now we can use Express to run

In order to do so we use the following method:

app.get("/", (req, res ) => {
return res.send(navbarPage + frontPagePage + footerPage)
});

This method allows us to access our frontpage when we request to route to "/"
